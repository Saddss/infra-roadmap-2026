# STAR 项目故事

把 M1-M5 的产出物全部串成 5 个项目故事。每个故事用 STAR 法则讲：

- **S** ituation：当时的背景 / 痛点
- **T** ask：你的目标
- **A** ction：你做了什么（这是面试官最关心的）
- **R** esult：结果（一定要量化）

每个故事必须能扛 30 分钟追问。

## 5 个项目（按推荐顺序）

1. `01-mini-llama.md` — 6 个月跨度的"基础重学"项目
2. `02-triton-flashattn.md` — CUDA/Triton 硬技术展示
3. `03-vllm-source-deepdive.md` — vLLM 源码精读 + PR
4. `04-inference-bench-report.md` — 推理框架横向 benchmark
5. `05-arch-evolution-blog.md` — 架构演进综述（V4 / Qwen3.6）

## 写作要点

- **每个故事都要有一个"亮点数据"**：
  - 项目 1: "实现了 MHA/GQA/MLA 三种注意力，KV Cache 减少 92%"
  - 项目 2: "Triton FlashAttention 在 8K seq_len 下比 naive PyTorch 快 X×"
  - 项目 3: "向 vLLM 主仓库提交 X 个 PR，Y 个已 merge"
  - 项目 4: "对比 vLLM/SGLang 后发现 SGLang 在 Agent 场景吞吐高 X%"
  - 项目 5: "综述博客在知乎获得 X 赞 / Y 评论"

- **每个故事都要有一个"踩坑细节"**：让面试官知道你真做过

- **每个故事都要能反向追问**：被问到深处不慌

## 时间投入

W24 周五一天，每个故事写 1.5 小时，写完通读一遍删掉所有"应该"、"大概"、"可能"。
